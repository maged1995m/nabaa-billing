# نبع — نظام فوترة المشتركين (نسخة أوفلاين كاملة)

تطبيق ويب مستقل بالكامل (بدون سيرفر PHP أو قاعدة بيانات MySQL، ولا يحتاج أي سيرفر محلي إطلاقاً)
لإدارة اشتراكات المياه/الكهرباء، إصدار الفواتير الشهرية تلقائياً من قراءات العدادات، وتحصيل الدفعات.

```
فرق القراءة        = القراءة الحالية − القراءة السابقة
إجمالي الاستهلاك   = فرق القراءة + الوحدات التقديرية
قيمة الاستهلاك     = إجمالي الاستهلاك × سعر الوحدة
المبلغ المستحق     = قيمة الاستهلاك + الرسوم الشهرية + المتأخرات السابقة − التسوية
المبلغ المتبقي     = المبلغ المستحق − المبلغ المدفوع
```

---

## 🗄️ أين تُحفظ بياناتك فعلياً؟ (مهم)

التطبيق يختار مكان الحفظ تلقائياً حسب طريقة تشغيله:

| طريقة التشغيل | مكان الحفظ | هل يُمسح مع "مسح بيانات المتصفح"؟ |
|---|---|---|
| **تطبيق مثبَّت (APK) بعد التحويل** | ملف حقيقي داخل مساحة تخزين التطبيق الداخلية على الجهاز (`Directory.Data` عبر Capacitor Filesystem) | **لا** |
| فتح `index.html` مباشرة في متصفح عادي (قبل التحويل، للاختبار) | `localStorage` الخاص بالمتصفح | نعم، إذا مسح المستخدم بيانات الموقع يدوياً |
| المعاينة داخل المحادثة | الذاكرة المؤقتة فقط | تُفرَّغ بمجرد إغلاق المعاينة (طبيعي لأي معاينة تجريبية) |

بمجرد تحويل التطبيق إلى APK، ستُحفظ بياناتك كملف `nabaa_billing_db.json` داخل مساحة التطبيق
الخاصة على الهاتف نفسه — **وليس في ذاكرة المتصفح** — تماماً كأي تطبيق أندرويد عادي، ولا تتأثر
بمسح "بيانات المتصفح" لأن التطبيق أصلاً لا يعمل داخل متصفح بعد التحويل، بل داخل غلاف Android حقيقي.

**التطبيق لا يحتاج أي سيرفر محلي (لا PHP ولا غيره) إطلاقاً** — هذا فرق جوهري عن أي نظام PHP قديم؛
كل شيء HTML/JS يعمل مباشرة داخل غلاف Android عبر Capacitor.

## 1) التشغيل الفوري كتطبيق ويب (للاختبار قبل التحويل)

```bash
npm install       # يجهّز أيضاً ملفات Capacitor وExcel تلقائياً عبر postinstall
npm run serve     # يفتح على http://localhost:8080
```

في هذا الوضع (متصفح عادي) يُستخدم `localStorage` تلقائياً كحفظ مؤقت، فقط ريثما تحوّله لتطبيق حقيقي.

---

## 2) الحصول على ملف APK حقيقي — بدون تثبيت أي برنامج (الطريقة الموصى بها)

لا تحتاج Android Studio على جهازك إطلاقاً. المشروع يحتوي ملف أتمتة جاهز (`.github/workflows/build-apk.yml`)
يبني لك APK حقيقي "في السحابة" مجاناً عبر GitHub Actions. الخطوات:

1. أنشئ حساب [GitHub](https://github.com) مجاني إن لم يكن لديك واحد.
2. أنشئ مستودع (Repository) جديد فارغ، وارفع له كل ملفات هذا المشروع (يمكنك السحب والإفلات
   مباشرة من موقع GitHub في المتصفح دون أي أوامر Git، عبر زر "Add file → Upload files").
3. من تبويب **Actions** أعلى المستودع، ستجد "بناء APK" — اضغط **Run workflow**.
4. انتظر 3-5 دقائق حتى ينتهي (دائرة خضراء ✓)، ثم افتح تفاصيل التشغيل وحمّل الملف
   المرفق باسم **nabaa-billing-apk** أسفل الصفحة (سيكون بصيغة zip يحتوي `app-debug.apk`).
5. انسخ `app-debug.apk` لهاتفك، فعّل "السماح بالتثبيت من مصادر غير معروفة"، وثبّته.

بهذه الطريقة يُبنى APK حقيقي فعلياً دون تثبيت أي أداة على حاسوبك.

> ملاحظة: هذا ينتج نسخة **Debug** (تجريبية) صالحة للاستخدام والتثبيت الشخصي مباشرة. إن رغبت
> بنشرها على متجر Google Play لاحقاً، ستحتاج توقيعها بمفتاح Release — راجع القسم التالي.

## 3) البديل: البناء محلياً عبر Android Studio (لمن يفضّل ذلك أو يريد نسخة موقّعة)

### المتطلبات
- [Node.js](https://nodejs.org) (18 فأعلى)
- [Android Studio](https://developer.android.com/studio) مع Android SDK

### الخطوات

```bash
# 1. ثبّت الاعتماديات (Capacitor + Excel، وتُنسخ ملفاتها تلقائياً إلى js/vendor/)
npm install

# 2. أضف منصة أندرويد
npx cap add android

# 3. زامن الملفات مع مشروع أندرويد
npx cap sync

# 4. افتح المشروع في Android Studio
npx cap open android
```

داخل Android Studio: **Build → Build Bundle(s) / APK(s) → Build APK(s)**، ثم ثبّت ملف
`app-debug.apk` الناتج على هاتفك (فعّل "السماح بالتثبيت من مصادر غير معروفة" أولاً).

بمجرد فتح التطبيق المثبَّت، سيكتشف تلقائياً أنه يعمل داخل تطبيق حقيقي (عبر فحص `Capacitor.isNativePlatform()`)
ويبدأ الحفظ في تخزين الجهاز الداخلي مباشرة — لا حاجة لأي إعداد إضافي منك.

> إن واجهت أي مشكلة تتعلق بامتداد `@capacitor/filesystem` أثناء `npx cap sync`، تأكد من تشغيل
> `npm install` أولاً (وليس فقط `npm run serve`) حتى تُنسخ ملفات `js/vendor/` المطلوبة.

### لإصدار نسخة موقّعة (للنشر الفعلي على المتاجر)
استخدم **Build → Generate Signed Bundle / APK** داخل Android Studio، وأنشئ مفتاح توقيع (Keystore)
خاص بك عند أول مرة، واحتفظ به بأمان.

## 4) الميزات المتوفرة

- **المشتركون**: بيانات كاملة + منطقة/مربع اختياري + بحث وفلترة
- **استيراد/تصدير Excel (xlsx) حقيقي**: قالب Excel جاهز للتعبئة، استيراد دفعة مشتركين، وتصدير
  كامل قائمة المشتركين إلى ملف Excel — عبر مكتبة SheetJS (تعمل أوفلاين بالكامل في نسخة الـ APK)
- **أنواع الاشتراك**، **المناطق والمربعات**
- **الدورات**: فتح/إغلاق مع ترحيل الأرصدة تلقائياً — وأي مشترك لم تُسجَّل قراءته عند الإغلاق
  تصدر له فاتورة تلقائية بنفس قراءته السابقة (استهلاك صفر) + الرسوم الشهرية + متأخراته
- **إدخال القراءات**: حساب فوري للفاتورة أثناء الكتابة
- **الفواتير والدفعات**: دفعات كاملة/جزئية، QR حقيقي أوفلاين على كل فاتورة
- **المتحصلون**: فريق تحصيل ميداني، وكل دفعة تُسجَّل باسم من استلمها
- **نظام مستخدمين وصلاحيات**: مدير النظام (كل شيء) / قارئ عدادات (القراءات فقط) / متحصل (الفواتير والدفعات فقط)
- **التقارير**: متأخرات، فواتير غير مسددة، استهلاك حسب الدورة، كشف متحصل
- **نسخ احتياطي/استيراد JSON** لكامل قاعدة البيانات (من شاشة الإعدادات) — مفيد كنسخة إضافية
  بجانب الحفظ التلقائي على الجهاز

### الحساب الافتراضي عند أول تشغيل
اسم الدخول: `admin` — الرقم السري: `1234` (غيّره من شاشة «المستخدمون»)

> ملاحظة أمنية: الرقم السري هو فصل صلاحيات بسيط على نفس الجهاز، وليس تشفيراً حقيقياً.
